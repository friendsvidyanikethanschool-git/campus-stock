Admin PIN

Optional. When set, deleting items, emptying trash, and restoring a backup will ask for this PIN first — a lightweight guard against accidental or casual changes on a shared device. This is not secure, server-verified authentication; anyone with the PIN (or who clears it) can bypass it. It's stored only on this device.

Low-stock alerts

Shows a banner on the Inventory dashboard whenever items are low or out of stock. You can also enable browser notifications, which fire while this tab is open (they won't wake a closed browser).

Bulk import (CSV)

Add many items at once instead of one-by-one. Columns expected: Name, Category, Location, SKU, Quantity, Unit, MinStock, UnitCost, Custodian, Description, Notes (header row required; unknown categories/locations are created automatically).

Backup & restore

This page keeps its data in Claude's artifact storage, shared with anyone who has the link — there's no local server or fixed IP address behind it, since it's a page running in your browser, not a hosted app. Use these buttons to keep an offline copy on your own computer, and restore from one if needed.

Want this reachable at a fixed address on your school network?
That needs an actual host, which is outside what a page like this can do on its own. The usual path: save this HTML file, serve it from a school computer with something like Python's http.server or Node's serve, and ask your network admin for a static (reserved) IP or DHCP reservation for that machine. From there, saving to a database on that server — instead of this page's built-in storage — would give you real local backups whenever the machine is on, internet or not.
